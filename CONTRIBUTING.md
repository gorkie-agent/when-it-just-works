# Contributing to "When It Just Works"

First of all: thank you for being here. You must have experienced the magic.

## Ways to contribute

### 📝 Share your "It Just Worked" story

Open an issue with the template:

```
**What were you building?**
[e.g., a REST API, a React app, a bash script to rename 300 files]

**Language/Framework?**
[e.g., Python, TypeScript, Go, raw HTML like a maniac]

**How long did you stare at the screen?**
[e.g., 3 seconds, 45 minutes, still staring]

**Did you tell anyone?**
[Yes/No, and I should have]
```

### 🐛 Report a bug in this repo

Wait. If you found a bug here... that's kind of ironic. Please open an issue anyway.

### ✨ Add to the Feelings list

Found a developer feeling that deserves a spot in the ranking? Submit a PR! The criteria are:
- It must be a genuinely delightful developer moment
- It must be something we've all felt
- Extra points if it involves snacks

## Code of Conduct

Be kind. Be humble. Remember that tomorrow your code will break in ways you can't imagine. Stay grounded.

## Pull Request Process

1. Fork the repo
2. Create a branch named something delightful (e.g., `added-coffee-feeling`)
3. Make your changes
4. Submit a PR
5. If the PR merges without conflicts on the first try... well, that's just beautiful, isn't it?

---

Thank you for celebrating the magic with us. 🎉
